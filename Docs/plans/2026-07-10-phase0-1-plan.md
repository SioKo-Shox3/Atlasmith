## v3 裁定事項(差分再レビューの残指摘への対応 — 裁定はオーケストレーター)

1. **恒等転写ゲートの一本化**(B2 残): 「bit 一致 or 1e-6」の選択式を廃止し、**単一ゲート = max abs diff ≤ 1e-6**。bit 一致は `(i+0.5)/W*W-0.5` の FP 往復丸めで非決定的になり得るため不採用。1e-6 は 8bit 1階調(≈3.9e-3)の 1/4000 で、検出対象(半テクセルずれ ≈1e-3)を確実に落とせる。
2. **一方向オラクルの assert 強化**(B3 残): 内部点集合の**非空 assert**+**内部点比率の下限**(初期案 50%、test-designer が実装前確定)を追加。
3. **面対応の contract を corner まで定義**(C8/新規): 対応表 = `face_map (M_new,) int64` + **corner 整列を構築時に正規化**(新面 i の corner k の旧頂点 = `old_faces[face_map[i], k]` が成立するよう置換を吸収)。テストは全面・全 corner の 3D 座標一致で検証。
4. **`bake_maps` の戻り値型を確定**(C9 残/新規): `BakeResult`(NamedTuple: `maps: dict[str, ndarray]` / `chart_coverage: ndarray bool` / `valid_mask: ndarray bool`)。
5. **`bake_maps` は対応表を知らない**(新規 blocker): 入力契約 =「`faces_new_uv` と `faces_old_uv` は行・corner 整列済み」。整列は呼び出し側(CLI/テスト)が `face_map` で実施。
6. **`MeshData` は公開・provisional に変更**(C10 残): 公開関数の型に現れる以上 internal は矛盾。「pre-1.0 はフィールド追加があり得る」を docstring に明記。repack 時の合成契約: `new.source_vertex = old.source_vertex[vmapping] if old.source_vertex is not None else vmapping`。
7. **公開高水準 API を追加**(新規 blocker): `rebake(input_path, output_path, *, texture_size=1024, padding_px=8) -> None`。CLI はこの薄いラッパ。公開 API は **5関数+CLI**。
8. **UV↔テクセル変換式を品質契約に明記**(新規 should-fix): テクセル (r,c) の中心 UV = `((c+0.5)/W, (r+0.5)/H)`、サンプリング座標 `x = u*W-0.5, y = v*H-0.5`。
9. **オラクル基準値側サンプラはテスト独立実装**(新規 should-fix): production の sampler を import しない。
10. **主ゲートは平滑テクスチャのみ**(新規 should-fix/N-3): per-face ID・非周期パターンは負の対照と恒等転写専用。シーム帯除外は**新旧両方**のチャート境界を対象。
11. **恒等転写の評価マスクは独立カバレッジの 1px 侵食(内部)**(N-4): 境界テクセルの tie-break 差によるフレーク防止。
12. その他: Step 1-2 許可パスに `tests/conftest.py` 追加(N-1)、CI に job env `UV_PYTHON: ${{ matrix.python }}` 明記(N-2)。

## v2 改訂概要

- **主ゲートを一方向オラクルへ変更**(B1): 同一 face+重心座標のサンプル点で「旧テクスチャ直接サンプル vs 新テクスチャサンプル」を一方向比較。往復比較は補助テストへ降格。
- **品質契約を実装前に固定**(B2/C12): data_range=1.0 float32・チャンネル一括MSE・sRGB非変換・alpha通常補間を規約化。閾値は test-designer が**実装前に**確定。
- **評価マスクの独立化**(B3): テスト側の素朴ラスタライザで独立生成。非空必須・カバレッジ率別assert・シーム帯分離。
- **pyproject に `[build-system]`+`version` 追加**(B4): backend は hatchling 推奨、技術選定記録+承認対象に追加(判断7)。
- **ミラー検査を `git diff --no-index --exit-code` へ全置換**(B5: PowerShell の `diff` は Compare-Object で内容比較にならない)。
- **API 境界の是正**(C8/C10/C11): 面対応表の明示構築+テスト、`MeshData` に weld mapping 追加、xatlas 素朴統合ヘルパは private 降格。
- **実装細部の是正**(C9/S3): padding を pack/bake で単一ソース同期、ガター膨張は非循環シフト、UV三角形の両巻き順対応。
- **fixture/CLI テスト強化**(C13)、**CI 強化**(C14: `--locked`・python 明示・SHA pin)、**コミット3の CLI を `--help` スタブに縮小+ロールバック方針書き直し**(C15)。
- **OBJ 忠実度問題時は停止してユーザー報告**(B7)、Step 1-3 許可パス補完(B6)、requires-python は 3.11 下限で統一し coding-style.md を同期(S5)。

## Phase 1 — テクスチャ焼き直し(⑤)の完成

前提: 実装前に `test-designer`(最上位)が本計画のテスト戦略を具体化し、**閾値を実装前に確定する**(B2 — fixture 実測後の追認は不可。確定後の閾値変更は計画レベルの正当化を必要とし、黙って緩めない)。

### Step 1-1: bake コア(転写+ガター+複数マップ)+ metrics【コミット7】

- **目的/挙動変化**: 「新旧メッシュの面対応が既知」という条件下で、旧UV/旧テクスチャ→新UVへの焼き直しが純 numpy で動く。
- **ファイル(許可)**:
  - `src/atlasmith/bake/__init__.py`、`src/atlasmith/bake/transfer.py` — 公開 API: `bake_maps(faces_new_uv, uv_new, faces_old_uv, uv_old, maps_old: dict[str, ndarray], *, size: tuple[int, int], padding_px: int) -> BakeResult`。**`BakeResult` = NamedTuple(`maps: dict[str, ndarray]` / `chart_coverage: ndarray bool` / `valid_mask: ndarray bool`)**(裁定4 — チャートカバレッジと膨張後有効マスクは別フィールド、C9)。**入力契約(裁定5): `faces_new_uv` と `faces_old_uv` は行・corner 整列済み**(整列は呼び出し側が face_map で実施 — bake は対応表を知らない)。
  - `src/atlasmith/metrics/__init__.py`、`src/atlasmith/metrics/quality.py` — `masked_psnr(a, b, mask) -> float`(**マスクが空なら例外** — B3)。
  - `tests/test_bake_transfer.py`、`tests/test_metrics.py`、`tests/conftest.py` への fixture 追加。
- **ファイル(禁止)**: bake/ から `trimesh`・`xatlas`・`PIL` の import 禁止(numpy+標準ライブラリのみ)。io/pack/`pyproject.toml`/Docs の変更禁止。
- **アルゴリズム方針**(依存を増やさず実装可能):
  1. **ラスタライズ**: 新UVをテクセル座標に変換(変換式は品質契約 — 裁定8)し、三角形ごとに BBox 内テクセル中心をエッジ関数で内外判定+重心座標算出。**UV三角形の符号付き面積で巻き順を正規化(または両符号許容)**(S3)。共有エッジの二重判定は tie-break 規約(top-left 規則等)で排除。三角形単位の numpy ベクトル化。出力: テクセル→(面ID, 重心座標)。
  2. **対応付け**: 行・corner 整列済み入力(裁定5)の下で、同一行・同一重心座標で旧UV頂点を補間 → 仕様の「テクセル3D位置→旧UV」と厳密に一致。対応付け関数は独立させ、Phase 2(トポロジ非対応時の最近傍表面クエリ)で差し替え可能にする。
  3. **サンプリング**: 旧テクスチャをバイリニア補間。規約は「横断規約」の品質契約に従う(clamp 既定・alpha 通常補間・sRGB 非変換)。対応付けは全マップで1回だけ計算し使い回す。
  4. **ガター膨張**: チャートカバレッジマスクを `padding_px` 回、3x3 近傍の最近傍値伝播で反復膨張。**`np.roll` は使わない(反対端へ回り込む)— 境界非循環のスライスシフトで実装**(C9)。
- **既知の限界(README/architecture.md に明記、判断6)**: タンジェント空間法線マップは UV 変更で基底が変わるため画素転写だけでは照明的に不正確。機構としては転写し、CLI が警告を出す。品質保証対象は basecolor+スカラー map に限定(C12)。
- **テスト戦略**(ラベルは v1 踏襲。**(b) は Step 1-2 の再展開後検証**(N2)):
  - (a) **恒等転写**: 新UV=旧UV、同解像度 → 評価マスク内で **max abs diff ≤ 1e-6(単一ゲート — 裁定1)**。評価マスクは**テスト側の素朴ラスタライザ**(per-texel 点内判定の遅い独立実装。小解像度で可)で生成した独立カバレッジの **1px 侵食(内部)**(B3/裁定11)。非空を必須、カバレッジ率(下限は test-designer が事前確定)を別 assert。per-face ID・非周期パターンを使用可(恒等はシーム誤差が出ないため — 裁定10)。
  - (c) **ガター**: `padding_px=N` で、カバレッジ境界から N px 以内の外側テクセルが全て有効値かつ近傍チャート値と一致。`padding_px=0` との差分が非ゼロ(膨張が動いた証拠)。画像端のチャートで回り込み汚染が無いこと(C9 の検証)。
  - (d) **複数マップ**: basecolor+合成ラフネスの2マップ一括 → 有効マスク完全一致、各マップが個別焼きと bit 一致。
  - **負の対照(必須)**: 旧UVを 0.1 シフトした誤対応・チャンネル入替・V反転が閾値を大きく下回ること(メトリクスとテストが「落ちられる」証明。per-face ID / 非周期パターン fixture を使用 — C13)。
- **検証**: `uv run pytest tests/test_bake_transfer.py tests/test_metrics.py -v`(収集件数付き)+ 全ゲート3点。1024px×トーラスで数秒オーダーの実測(厳密な性能ゲートは設けない)。
- **リスク**: 中(数値アルゴリズムの正しさ)。縮退三角形・共有エッジは tie-break で対処。ロールバック: bake/・metrics/ は他モジュール未参照のため revert 安全。
- **コミット可能条件**: (a)(c)(d)+負の対照が green。メッセージ例: 「UV間テクスチャ転写・ガター膨張・PSNR計測を追加する」。

### Step 1-2: xatlas 素朴展開(internal)+ 一方向オラクル検証【コミット8】

  - `tests/test_bake_oracle.py`、`tests/conftest.py` への fixture/ヘルパ追加(N-1/裁定12)。
- **ファイル(禁止)**: bake/ の変更禁止(境界の証明のため)。io/`pyproject.toml`/Docs の変更禁止。
- **一方向オラクルの枠組み**(B1 — 詳細設計は test-designer へ引き継ぎ。方式と合格条件の枠組みは以下で固定):
  - **サンプル点生成**: 旧メッシュの各面につき、シード固定の一様重心座標サンプル K 点(+面重心)。
  - **分類**: 各点の新UV位置が独立生成カバレッジ(テスト側ラスタライザ — B3)のチャート境界、**および旧UV側のチャート境界**(裁定10)から `seam_margin` テクセル以内なら「シーム帯」、それ以外は「内部」。**内部点集合の非空 assert + 内部点比率 ≥ 下限(初期案 50%、test-designer が実装前確定 — 裁定2)**。内部点が主ゲート、シーム帯は別テスト(緩い閾値 or 外れ値率上限 — test-designer が事前確定)。

## 横断規約(全ステップ共通)

- **依存方向(Step 0-6 で architecture.md に記録)**: `cli → io/pack/bake/metrics`、`pack → types (+xatlas)`、`io → types (+trimesh/PIL)`、`bake/metrics → numpy のみ`。逆向き import 禁止。`types.py` は無依存。
- **公開 API(C10/C11/裁定6・7)**: `load_mesh` / `save_mesh` / `bake_maps` / `masked_psnr` / `rebake` の**5関数**+CLI エントリポイント。`MeshData` は公開・**provisional**(pre-1.0 はフィールド追加あり得る — docstring 明記)。`_naive_unwrap_and_pack` は internal。`SegmentationBackend`/`SeamStrategy` は architecture.md 上の設計記録のみ(コード化しない)。
- **IR・サンプリング品質契約(B2/C12/裁定8 — 実装とテストの共通仕様)**: 画素は float32 [0,1]・channels last・data_range=1.0。PSNR の MSE は全チャンネル・全評価点一括。**sRGB は Phase 1 では変換しない**(線形配列として byte 保存的に扱う)。alpha は通常チャンネルとして補間(premultiply しない)。サンプラの境界規約は clamp 既定(wrap はオプション)。maps の V 方向は「row 0 = 画像上端 = V=0」(glTF 規約、OBJ は io 層で吸収)。**UV↔テクセル変換式: テクセル (r,c) の中心 UV = `((c+0.5)/W, (r+0.5)/H)`、サンプリング座標 `x = u*W-0.5, y = v*H-0.5`**(裁定8)。
- **スレッド/並行性**: 全段シングルスレッド前提。`MeshData` は値渡し、引数の破壊的変更禁止。
- **実行するゲート**(Step 0-3 以降は実ゲート): `uv run ruff format --check .` / `uv run ruff check .` / `uv run pytest`(常に収集件数を報告)。型チェッカーは未選定のまま(本計画スコープ外)。
- **コーディング規約**: LF+UTF-8(BOMなし)、公開関数に型ヒント必須、モジュール冒頭に目的ヘッダ、WHY コメント(coding-style.md 準拠)。ミラー検査は常に `git diff --no-index --exit-code -- CLAUDE.md AGENTS.md`(B5)。
- **リスクレベル総括**: 高リスク該当 = Step 0-2/0-3(依存追加)、0-4(公開 API・IR 初期確定)、0-6(ワークフロー資産)。いずれも計画レビュー一次+Codex 二次必須(本計画は v1〜v3 で実施済み)。Phase 1 は中リスク(数値正しさはテスト+一方向オラクルで担保)。
- **ロールバック方針(C15 で書き直し)**: revert は**最新コミットから逆順**に行う(任意ステップ単独 revert は保証しない — 後続が前方成果に依存するため)。未 push 区間は `git reset`/rebase 可、push 済みは revert コミットで対処。**uv 本体・`uv python install` 等リポジトリ外状態は revert の対象外**(削除手順は technique-package-manager.md に記録)。
